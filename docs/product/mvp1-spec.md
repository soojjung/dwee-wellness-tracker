## 1. 프로젝트 개요

### 1-1. 프로젝트명

여성 맞춤 컨디션 & 생리 주기 기록 앱 (MVP 1)

### 1-2. 한 줄 소개

생리 주기와 오늘의 몸 상태를 가볍게 기록하고, 내 패턴을 한눈에 확인할 수 있는 여성 라이프 로그 앱

### 1-3. MVP 1의 핵심 방향

MVP 1은 **"생리 주기 기록"**을 중심으로 시작한다.

초기 유저가 부담 없이 사용할 수 있도록, 다이어트/체중감량 기능은 전면에 내세우지 않고 **"컨디션 기록"** 중심으로 설계한다.

- **메인 가치:** 생리/배란/예상일 추적
- **보조 가치:** 몸 상태/기분/증상 기록
- **제외:** 칼로리 계산, 강한 체중 감량 유도, 복잡한 AI 분석

---

## 2. 문제 정의

### 2-1. 사용자 문제

기존 여성 건강 앱들은 아래와 같은 문제가 있음:

1. **너무 복잡함** — 배란, 임신, 영양, 운동, 성생활, 약 복용 등 초기 화면부터 정보 과다
2. **다이어트 중심 앱은 진입장벽이 있음** — 생리만 기록하고 싶은 사용자에게는 부담스럽고 피로함
3. **한국 사용자 기준 UX가 아쉬움** — 번역투 UI, 복잡한 입력, 과한 기능 나열
4. **내 몸의 패턴을 직관적으로 보기 어려움** — 생리 전 감정 변화, 붓기, 통증, 식욕 등 패턴이 축적되어도 한눈에 보기 어려움

---

## 3. MVP 1 목표

### 3-1. 제품 목표

사용자가 **매일 10초 안에** 자신의 상태를 기록하고, **한 달만 사용해도 "내 패턴"이 보이는 경험**을 제공한다.

### 3-2. 핵심 목표 지표 (초기)

- 회원가입 후 **첫 기록 완료율 70%+**
- 7일 내 **재방문율 35%+**
- 주 3회 이상 기록하는 **활성 유저 비율 25%+**
- 첫 달 내 **생리 예측 정확도 체감 만족도 확보**
- 앱 삭제 전 최소 2주 사용 유도

---

## 4. 타겟 사용자

### 4-1. 핵심 타겟

**20~30대 여성**

- 생리 주기 기록이 필요하지만 기존 앱이 부담스러운 사람
- 생리 전/중 컨디션 변화를 파악하고 싶은 사람
- 다이어트보다는 **"몸 상태 관리"**에 관심 있는 사람
- 심플하고 감성적인 UI를 선호하는 사람

### 4-2. 페르소나

#### 페르소나 A — "기록은 하고 싶지만 복잡한 건 싫은 직장인"

- 28세
- 생리 예정일만 대략 알고 싶음
- PMS(붓기, 예민함, 식욕 증가)가 심함
- 복잡한 건강 앱은 오래 못 씀

#### 페르소나 B — "내 몸의 패턴을 알고 싶은 웰니스 관심층"

- 25세
- 컨디션, 기분, 피부, 식욕 변화를 기록하고 싶음
- 다이어트 앱은 부담스럽고 강박적이라 싫음
- 감성적인 기록형 UX를 좋아함

---

## 5. MVP 1 핵심 가치 제안 (Value Proposition)

1. 생리 예정일을 쉽게 파악
2. 오늘의 컨디션을 간단히 기록
3. 생리 전/중 몸 상태 패턴을 확인
4. 다이어트 강요 없이 편안한 웰니스 경험 제공

---

## 6. MVP 1 기능 범위 (Scope)

### 6-1. 포함 기능 (Must Have)

#### 1) 생리 주기 기록

- 생리 시작일 입력
- 생리 종료일 입력 (선택)
- 생리 기간 자동 계산
- 평균 주기 계산
- 다음 생리 예정일 예측
- 캘린더에서 생리일/예정일 표시

**핵심 이유:** 앱의 존재 이유이자 가장 강한 리텐션 포인트.

#### 2) 오늘의 컨디션 기록 (1일 1회 간편 입력)

사용자는 오늘 상태를 아주 빠르게 기록할 수 있음.

기록 항목 예시:

- 기분 (좋음 / 보통 / 예민 / 우울)
- 에너지 (높음 / 보통 / 낮음)
- 통증 (없음 / 약함 / 심함)
- 붓기 (없음 / 있음)
- 식욕 (보통 / 증가 / 감소)
- 피부 상태 (좋음 / 트러블)
- 메모 (선택)

**핵심 이유:** 생리 앱 이상의 "내 몸 패턴 인사이트"를 만들기 위한 최소 데이터.

#### 3) 생리 단계 기반 상태 표시

홈 화면에서 현재 상태를 보여줌.

예시:

- 생리 중
- 생리 예정 3일 전
- 배란 추정기
- 안정기 (주기 중간)

> ⚠️ MVP 1에서는 "의학적 진단"처럼 보이지 않도록 **"추정 / 예상 / 참고용"** 표현 사용.
> (`CLAUDE.md` 도메인 표현 규칙 및 `.claude/rules/health-copy.md` 참조)

#### 4) 캘린더 / 월간 기록 보기

- 월간 캘린더 화면
- 생리 기록 표시
- 오늘 기록 여부 표시
- 선택 날짜의 컨디션 요약 확인

#### 5) 간단한 패턴 인사이트

최소 수준의 통계 제공.

예시:

- "최근 3개월 평균 주기: 29일"
- "생리 전 3일 동안 식욕 증가 기록이 많아요"
- "생리 1~2일차에 통증 기록이 자주 있어요"

**MVP 1에서는 AI 대신 rule-based**로 충분.

#### 6) 알림 기능 (선택 가능)

- 생리 예정일 알림
- 기록 리마인드 알림 (매일 저녁 1회 or 사용자 지정)
- 생리 시작일 기록 유도

> ⚠️ `CLAUDE.md` 명시적 제외에 "실제 푸시" 항목이 있어, MVP1 구현 단계에서는
> 알림 UX 토글/설정만 둔 mock 형태로 갈지 별도 결정 필요.
